## 这是什么 / What This Is

**中文**

PCR-QR 是一个用于诊断和干预公共社会问题的双层分析框架。

- **外层 PCR(公共结构层)**:政策(Policy)— 文明意识(Consciousness)— 生活规律(Routine)
- **内层 QR(经验显影层)**:感质(Qualia)— 节律(Rhythm)
- **接口**:*C 层的实际运行机制,是感质在节律中显影*

它解决一个普遍的诊断盲区:为什么很多公共议题(心理健康、性别暴力、气候、AI、教育)都看到话语在升温——就医率上升、报案率上升、信息量增加——但结构停滞依旧?单层政策分析在这种情况下倾向于得出"结构性问题需要结构性解决"的悲观结论。PCR-QR 提供另一个诊断角度:**伪进展陷阱**——升温被锁在错误的显影通道里。

**English**

PCR-QR is a dual-layer diagnostic framework for analyzing and intervening in public social problems.

- **Outer layer PCR (Public Structure)**: Policy — Consciousness — Routine
- **Inner layer QR (Experiential Manifestation)**: Qualia — Rhythm
- **Interface**: *the C-layer's actual operating mechanism is qualia manifesting through rhythm*

It addresses a common diagnostic blind spot: why do many public issues (mental health, gender violence, climate, AI, education) show numerical heating — rising treatment rates, reporting rates, information volume — while structural stasis persists? Single-layer policy analysis tends to conclude "structural problems require structural solutions" — true but actionably empty. PCR-QR offers another angle: **pseudo-progress traps** — heating locked into the wrong manifestation channel.

## 核心命题 / Core Proposition

**接口命题 / Interface Proposition**:

> *PCR 中"意识层"(C)的实际运行机制,是感质在节律中显影。*
>
> *The C-layer's actual operating mechanism is qualia manifesting through rhythm.*

这一步把 C 层从抽象"意识"变量转化为可操作诊断的对象。公共话语包含的不是"一般观念",而是已经通过某种公共采样节律成功显影的感质。**当感质存在但无法显影**——因为没有公共通道匹配其时间结构——话语停滞,无论增加多少信息都无效。

This single move converts the C-layer from an abstract "consciousness" variable into something operationally diagnosable. What public discourse contains is not "ideas in general" but qualia successfully manifested through some public sampling rhythm. **Where qualia exists but cannot manifest** — because no public channel matches its temporal structure — discourse stalls regardless of how much information is added.

### 在自己的领域应用框架 / Applying the Framework to Your Domain

跑诊断流程的五步:

1. **PCR 三元定位** — 你领域里 P / C / R 各自处于什么状态?
2. **QR 显影诊断** — 感质密度、分布、配置如何?节律错配的大小和类型?
3. **主导变量识别(双层)** — PCR 层主导变量是什么?QR 层呢?合成判断是什么?
4. **级联路径设计** — 哪些路径不可行(为什么)?哪一条最可行?
5. **干预点选择** — 哪个 Phase 杠杆最大?具体干预点是什么?

Run the five-step diagnostic procedure:

1. **PCR Triadic Mapping** — What state is each P / C / R in?
2. **QR Manifestation Diagnosis** — What's the qualia density, distribution, configuration? What's the rhythm mismatch magnitude and type?
3. **Dominant Variable Identification (dual-layer)** — Dominant at PCR? At QR? Synthesis?
4. **Cascade Path Design** — Which paths are infeasible (why)? Which is most feasible?
5. **Intervention Point Selection** — Which Phase has highest leverage? What are the concrete intervention points?

## v2.0 理论收获 / v2.0 Theoretical Gains

通过五案例加压测试,框架从 v1.1 自然演化出三个新概念体系:

Through stress-testing across five cases, the framework naturally evolved three new conceptual systems from v1.1:

### 1. 三类瓶颈结构 / Three-Tier Bottleneck Typology

```
通道遮蔽型 / Channel-occlusion    ← 5.1, 5.2
节律错配型 / Rhythm-mismatch      ← 5.3
复合型     / Compound             ← 5.4, 5.5
```

### 2. 三种干预工程 / Three Differentiated Engineering Types

| 工程 / Engineering | 主导案例 / Lead Case | 实践范例 / Practice Reference |
|---|---|---|
| 通道工程 / Channel Engineering | 5.1, 5.2 | 1960-70s 女权意识提升小组 / consciousness-raising groups |
| 节律工程 / Rhythm Engineering | 5.3 | 强制性金融披露 / mandatory financial disclosure |
| 感质工程 / Qualia Engineering | 5.4 | 沉浸艺术与互动设计 / immersive art & interactive design |

### 3. 四种感质密度结构 / Four Qualia Density Configurations

```
均匀高 / Uniform-high           ← 5.1
不均高 / Uneven-high            ← 5.2
尺度切分 / Scale-split          ← 5.3
类型切分 / Type-split           ← 5.4
两极分化 / Polarized            ← 5.5
```

### 4. 诚实失败边界 / Honest Failure Boundaries

v2.0 第一次承认有些问题(尤其气候)在可用决策节律结构内可能**数学不可解**。这是特性,不是缺陷——一个不能承认边界的框架不是真正的诊断工具。

v2.0 first admits that some problems (notably climate) may be **mathematically unsolvable** within available decision-rhythm structures. This is a feature, not a defect — a framework that cannot admit boundaries is not a real diagnostic tool.
