03 / Operational Procedure

PCR-QR v3.0 运行流程

v2.0 是双层诊断；v3.0 是运行协议。它把每一个公共问题送进同一条可审计流程。

v2.0 is a dual-layer diagnosis; v3.0 is an operational protocol. Each public problem enters the same auditable procedure.

Input / 输入选择一个公共问题，保留原始语言，不急着解释。
Naming Audit / 命名审计问题被什么词命名？谁拥有命名权？哪个词正在终止理解？
PCR Mapping / 公共结构映射政策、文明意识、生活规律三层如何耦合？
QR Manifestation / 感质节律显影问题在身体中是什么质地？被什么节律采样？
Operation Layer / 四层操作命名、加压、滑义、相变，找出真正瓶颈。
Engineering Choice / 工程选择通道工程、节律工程、感质工程，或三者组合。
Feedback / 反馈再命名现实输出反击语言，语言重新命名，系统继续生成。
v3.0 Formula: Input → Naming → PCR → QR → Pressure → Drift → Phase → Engineering → Feedback → Re-naming
04 / PCR-QR Core

双层结构保留，但升级为接口

PCR-QR 的原核心保留：外层结构与内层经验通过一个明确接口相连。

The original PCR-QR core remains: outer public structure and inner embodied experience are connected through one explicit interface.

P

Policy / 政策

法律、制度、预算、执行机制。问：什么被正式允许或禁止？

Law, institutions, budget, enforcement. What is formally allowed or prohibited?

C

Consciousness / 文明意识

什么算问题，谁能说话，什么语言被承认。

What counts as a problem, who can speak, which language is recognized.

R

Routine / 生活规律

工作、家庭、消费、居住如何无意中重复生产问题。

How work, family, consumption, and residence reproduce the problem.

Q

Qualia / 感质

问题在身体里的纹理：强度、分布、配置。

The bodily texture of a problem: intensity, distribution, configuration.

R

Rhythm / 节律

感质被采样和显影的时间结构：连续/事件、快/慢、多层叠加。

The temporal structure through which qualia is sampled and manifested.

Interface: C = Qualia × Rhythm / 文明意识层的实际运行机制，是感质在节律中显影。
